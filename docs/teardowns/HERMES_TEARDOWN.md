# Hermes teardown — what to port, fix, and avoid (2026-06-11)

Implementation-level mining of github.com/NousResearch/hermes-agent (MIT, Python).
Verdicts drive docs/FEATURE_PARITY_PLAN.md.

| Subsystem | Verdict | Core lesson |
|---|---|---|
| Skills-from-experience | **port-with-fixes** (our marquee) | Substrate is proven: SKILL.md (agentskills.io) + sidecar telemetry + curator GC + 3-level progressive disclosure + patch-over-create preference. Two structural holes WE fix: (a) no correctness mechanism — agent is author/executor/inspector of its own skills (#25833) → Muster: extraction emits CandidateSkill into quarantine; promotion ONLY through our eval gate; (b) inject-ALL index costs 10-15K tokens/call at 243 skills (#22620) → top-K budgeted injection (~2K cap). Extractor runs as tool-whitelisted child (their #15204 P1: unsandboxed reviewer acted as host). Stamp skill@version into token ledger per run (#11692 receipts). Counter-trigger (10 tool-iters) is fine. Their offline arm (DSPy/GEPA evolution emitting PRs) = our evolve loop's future. |
| Session search | **port-concept near-verbatim** (cleanest subsystem) | SQLite FTS5 (unicode61 + trigram), trigger-maintained; ONE tool with 4 arg-inferred shapes (discover/scroll/read/browse); zero LLM in search path (~20ms); async 3-7-word session titles make results scannable. Bake in: single-writer through gateway (their #5563 corruption = concurrent writers), continuation = summary handoff never full replay (1.9M-token waste), token_count on every message row, merge with our token ledger (one store). |
| Context management | **port-with-fixes — implement #14948 (they refused)** | Their 3-layer creation-time defense (per-tool truncation → persist oversized to file with fetch-by-id stub → per-turn cap) is good; their reactive, history-MUTATING compaction is the flaw, and their "[REFERENCE ONLY]" prefix demoted MEMORY.md so the model ignored it (#17251). Muster: immutable transcript + pure render(transcript, budget) per call — last N tool results verbatim, older → deterministic stubs with result_fetch(id); LLM summary only if stubbed transcript still exceeds budget; summary framing scoped to "earlier turns", placed BELOW memory/skill blocks. Benchmarked 57-82% token reduction at 20-50 tool calls. Highest token-ROI item in both studies. |
| Tool registry | **port-concept; avoid integration sprawl** | Declarative ToolEntry (schema, handler, availability fn w/ TTL cache, requiresEnv, per-tool result cap) + composable toolsets + webhook-safe subset. Build ~14 first-party tools (terminal/file quad/web pair/browser triad/delegate/cron/memory/session_search/send_message); the 7-Spotify-tools tail is what MCP is for. Typed ToolResult, not strings. |
| Scheduler | **port-concept** | No NL parser exists — the tool description teaches the LLM to emit one of 4 deterministic grammars; code validates (croniter). At-most-once: next_run_at advanced BEFORE execution; overdue fast-forwarded, never burst. Fresh agent per job, cronjob/messaging tools force-disabled, [SILENT] marker, MEDIA: tags. Muster addition: per-job maxUsdPerRun from ledger, auto-pause. |
| Subagent delegation | **port-with-fixes** | Proven: depth-1 default, CHILD_BLOCKED frozenset (delegate/clarify/memory-write/send_message), 600s timeout + stall watchdog, child cost folded into parent ledger tagged "subagent", file-conflict registry. Fix: sync ThreadPool(3) → Node async fan-out. Children REPORT learnings; parent's eval-gated extractor decides persistence. |
| Profiles | **port two tricks** | (1) home/ subdir isolates subprocess credentials (git/ssh/npm) per profile — can't leak; (2) clone semantics: config+memory+skills WITHOUT sessions. Keep our single-gateway multiplexing (their process-per-profile is Python-monolith workaround); separate SQLite file per profile (less write contention). |
| Desktop/gateway API | **port-concept** | ONE newline-delimited JSON-RPC 2.0 protocol identical over stdio and WS (TUI/desktop/web all consume it); integer contract-version handshake (DESKTOP_BACKEND_CONTRACT); WS auth via 30s single-use tickets minted over authenticated REST. Event vocabulary: message.chunk/stop, commentary, tool.start/complete(inlineDiff), approval.request, subagent.*. Muster addition: ledger.tick event = live cost in UI. Tauri sidecar, not Electron. |
| Voice/media | **port two ideas, late** | MEDIA:<path> inline tags keep agent output plain text; adapters render natively. Sentence-buffered streaming TTS (min 20 chars). Skip: 10-provider TTS matrix (need 2), live Discord voice, fal.ai lock-in decision is theirs not ours. |
| MCP client | **port-with-fixes** | Copy: config shape (stdio/HTTP transports, filtered env, include/exclude, OAuth), namespacing server__tool, background non-blocking discovery, circuit breaker (3 fail/60s). Their bugs = our test plan: per-server ISOLATED supervision (#34443 one server crashed all), mcp-session-id propagation (#20349), loud surfacing of silent handshake failures (#14113), MCP results forced through the same result-cap/persistence/ledger pipeline as built-ins (#44172), headless OAuth device-code (#44125). |

## Cross-cutting AVOID list (both teardowns)
inject-all skill indexes · mutate-history compaction · compaction framing that demotes
memory · multi-process SQLite writers · shared MCP failure domains · unsandboxed
self-review agents · first-party integration tool sprawl · runtime plugin installs ·
open skill registries · announce-push subagent results · reactive-only compaction
that needs a model call to unwedge.
