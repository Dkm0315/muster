# Muster site redesign — plan (vibe + breadth + content)

Two goals, learned from the Hermes (fluid/premium vibe) and OpenClaw (breadth/showcase) sites:
1. **Vibe:** fresh, fluid, 3D, Gen-Z-appealing — NOT amber/terminal. (Palette + 3D concept TBD from research.)
2. **Breadth + content:** structure like OpenClaw — show a large, alive surface and give people content to discover and share.

## Information architecture (the OpenClaw breadth pattern, applied honestly)

Muster does not have OpenClaw's 100+ first-party integrations — and shouldn't fake them.
Its honest breadth multiplier is **MCP (any of the thousands of MCP servers)** + **capability packs** + a **one-envelope surface contract**. Present that attractively.

1. **Hero** — fluid 3D, fresh palette, one signature moment.
2. **Connect anywhere (channels):** Telegram · Slack · Discord · WhatsApp · Google Chat · MS Teams · WebChat — "one governed envelope; add any surface with ~40 lines."
3. **Any model, your keys:** the real 20+ provider catalog as a grid — Claude (Fable 5), OpenAI, Gemini, Grok, Kimi, DeepSeek, Mistral, GLM, Perplexity, Qwen, Groq, Cerebras, OpenRouter, Together, Fireworks, Ollama, LM Studio, vLLM, SGLang, Codex CLI. "Cloud or local. Zero lock-in."
4. **Tools & integrations:** **MCP — connect any MCP server** (the breadth multiplier, with per-server isolation) · **Capability packs** (Frappe/ERPNext today) · **Flows** (durable, replayable automations) · the built-in tool registry.
5. **What makes it different (governance):** token ledger · scoped memory · eval-gated skills · `muster verify` · never-wedge compactor. Keep this tight — it's the moat.
6. **Proof:** the Token Waste Index — ~61% fewer tokens, live and reproducible.
7. **Showcase / recipes:** real, honest use cases (ERP ops copilot, cost-controlled fleets, air-gapped/BFSI, agent CI). Grows over time.
8. **Writing / blog:** content marketing for reach + SEO — "how scoped memory works," "the token waste index," "governed agents in production," teardowns. A `/blog` index + markdown posts.
9. **Get started + community:** install, docs, GitHub, capability-pack registry vibe ("share & discover packs").

## Content to seed (attract + breadth)
- Blog posts (3–5 to launch): Token Waste Index deep-dive; scoped memory & why leaks are a CI failure; eval-gated skills vs self-certified learning; one governed envelope for every chat app.
- Showcase entries: Frappe/ERPNext copilot; cost-controlled fleet; air-gapped deployment.
- A "packs" page positioning the capability-pack format as the community surface (Muster's honest answer to ClawHub).

## Honesty rules
- Real integrations only. MCP = "any MCP server" is true breadth; lean on it.
- No fabricated Spotify/Hue/Sonos-style first-party integrations we don't have.
- "Planned" clearly labeled where used.
